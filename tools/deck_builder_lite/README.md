# MOZ Deck Builder Lite

MOZ Deck Builder Lite は、「モンスターおぼえるぜ」用の問題ファイルをブラウザだけで作る軽量ツールです。

Pythonやインストール作業は不要です。`index.html` をブラウザで開くだけで使えます。

## 起動方法

`tools/deck_builder_lite/index.html` をブラウザで開きます。

GitHub Pagesで公開している場合は、次のようなURLからも開けます。

```text
https://cookingsaltsio.github.io/MOZ/tools/deck_builder_lite/
```

## 作れる問題

- 単語帳カード
- Cloze穴埋めカード
- `choice_exclude_words` による紛らわしい選択肢の除外
- `choice_group` による同義語グループ除外

画像問題は、現在のMOZ本体が非対応のため、このツールでも対応していません。

## 基本の画面

単語帳モードでは、画面上部にテキスト素材を置く2つのペインがあります。

- テキスト1: 主に元本文
- テキスト2: 単語帳モードでは解答側や対訳本文、穴埋めモードでは穴埋めdraft

画面下部でカードを作成・編集します。

穴埋めモードでは、単語帳用のdraft欄は表示されません。画面左側にテキスト1と穴埋めdraftが縦に並び、画面右側に作成済みカード一覧が表示されます。穴埋めdraftには行番号が付き、奇数行と偶数行で背景色が変わるので、どこまで作ったか確認しやすくなっています。

## テキスト素材の読み込み

次の方法で素材を入れられます。

- テキスト1読込
- テキスト2読込
- 直接貼り付け

対応ファイル:

- `.txt`
- `.html`
- `.htm`

HTML読み込み時は、`script`、`style`、`noscript`、`iframe`、`svg` などを除去し、本文をプレーンテキスト化します。

HTMLページによっては、広告、メニュー、フッターなどが混ざる場合があります。JavaScriptで本文を後から生成するページにも弱いです。その場合は、ブラウザ上で本文だけをコピーして貼り付けてください。

## 本文編集ロック

「本文編集ロック」がONのとき、本文ペインは選択しやすい状態になります。A/S/Dなどを押しても本文に文字が入力されにくくなります。

OFFにすると本文を直接編集できます。

## 青空文庫・特殊記号の整理

次のボタンで、青空文庫風のルビや注記、特殊な見えない文字を整理できます。

- テキスト1整理
- テキスト2整理
- 両方整理

主に次のようなものを整理します。

- `｜`
- `《ルビ》`
- `［＃注記］`
- ゼロ幅文字
- 余分な空白や連続改行

## 文分割整形

長い段落を、一文一行に分けられます。

- テキスト1整形
- テキスト2整形
- 両方整形

区切り文字は画面上で変更できます。

初期値:

```text
。！？!?．.：:
```

「本文から候補」を押すと、現在の本文に含まれる句読点から区切り文字候補を作ります。

## 検索

`Ctrl+F` で検索欄を開けます。

- 検索対象: テキスト1 / テキスト2 / 両方
- Enter: 次へ
- Shift+Enter: 前へ
- Esc: 閉じる

検索欄には、現在何件目かも表示されます。

## 単語帳カードの作り方

1. 作成モードを「単語帳」にします。
2. 本文の問題にしたい部分を選択して `A` を押します。
3. 本文の答えにしたい部分を選択して `S` を押します。
4. draft欄で必要なら文章を調整します。
5. `D` または「カード追加」でカード一覧へ追加します。

ショートカット:

- `A`: 選択中テキストを問題文draftへ追加
- `S`: 選択中テキストを解答文draftへ追加
- `D`: draftからカード追加
- `Z`: 直前操作を取消
- `X`: draftクリア
- `Ctrl+S`: 作業保存

## Cloze穴埋めカードの作り方

1. 作成モードを「穴埋め」にします。
2. テキスト1に元本文を貼り付けるか読み込みます。
3. テキスト1で、問題にしたい一文または段落を選択します。
4. `A` を押します。選択した文が、テキスト2の穴埋めdraftへ1問として追加されます。
5. 穴埋めdraftで、隠したい文字を選択します。
6. `1` を押すと `{{c1::...}}`、`2` を押すと `{{c2::...}}` の形に変換されます。
7. `D` を押すと、現在のdraft行が作成済みカード一覧へ登録されます。

穴埋めモードでは、上部の操作バーから「選択範囲をdraftへ追加」「draftの現在行をカード登録」「テキスト1の各行をdraftへ追加」も使えます。除外語やグループも同じ操作バーで指定できます。

穴埋めdraftは1行を1問として扱います。左側の番号はdraft内の問題番号です。登録した行はdraftから消え、右側の作成済みカード一覧へ移動します。

例:

```text
東京は日本の首都です。
```

テキスト1でこの文を選択して `A` を押すと、穴埋めdraftへ追加されます。その後、draft内で `東京` を選択して `1`、`日本` を選択して `2` を押すと次のようになります。

```text
{{c1::東京}}は{{c2::日本}}の首都です。
```

MOZ本体では、番号ごとにカード化されます。

```text
問題: ■■■は日本の首都です。
解答: 東京

問題: 東京は■■■の首都です。
解答: 日本
```

同じ番号を複数使うと、同時に隠されます。

```text
{{c1::東京}}は日本の首都で、昔は{{c1::江戸}}と呼ばれました。
```

この場合、解答は `東京 / 江戸` になります。

## Clozeカードの一括追加

穴埋めモードでは、整形済みの本文から各行をまとめて穴埋めdraftへ追加できます。

- テキスト1の各行をdraftへ追加
- 選択範囲を文分割してdraftへ追加

先に「テキスト整形」で一文一行にしておくと扱いやすいです。

一括追加した直後は、まだ `{{c1::...}}` が付いていないCloze候補です。穴埋めdraft内で隠す箇所を選択して `1` から `9` を押し、最後に `D` でカード一覧へ登録してください。

## 単語帳モードの使いどころ

大量の単語帳を作るだけなら、Excel、LibreOffice Calc、またはスクリプトで処理したほうが速い場合があります。

このツールの単語帳モードは、次のような手作業に向いています。

- 本文と対訳を見ながら、少量のカードを丁寧に抜き出す
- 長い説明文をその場で整えてからカード化する
- `choice_exclude_words` や `choice_group` をカードごとに手で調整する
- 既存の表計算ファイルにする前の下書きを作る

## 類義語・紛らわしい選択肢の設定

MOZでは、類義語や紛らわしい語を誤答選択肢から外せます。

このツールでは次の欄を設定できます。

- `choice_exclude_words`
- `choice_group`

### choice_exclude_words

指定した語を、そのカードの誤答選択肢から除外します。

例:

```text
一针一线、一星半点、微乎其微
```

区切りには、`、`、`,`、`;`、`/`、空白などを使えます。

### choice_group

同じグループIDを持つカード同士を、誤答選択肢から除外します。

例:

```text
sim_00001
```

同義語セットなどに使います。

## カード一覧

カード一覧では次の操作ができます。

- 問題文編集
- 解答文編集
- 除外語編集
- グループ編集
- 種別変更
- 削除
- 複製
- ドラッグアンドドロップ並べ替え
- 空行追加
- 一覧クリア

Clozeカードで穴埋め指定がまだないものは、薄い赤色で表示されます。そのままでは出力時にスキップされます。

## 出力形式

MOZが読み込める次の形式で出力できます。

- TSV
- CSV

出力される列:

```text
question
answer
choice_exclude_words
choice_group
```

単語帳カードでは、`question` が問題、`answer` が解答です。

Clozeカードでは、`question` に `{{c1::...}}` を含む本文が入り、`answer` は空になります。MOZ本体がCloze本文から解答を自動抽出します。

## MOZ本体への読み込ませ方

1. このツールでTSVまたはCSVを出力します。
2. MOZ本体を開きます。
3. ホーム画面の「デッキ管理」で出力したファイルを選びます。
4. 読み込まれたデッキを選択します。
5. 「全項目で出陣」などで確認します。

## 作業保存・復元

「作業保存」を押すと、作業中の状態を `.mozbuild.json` として保存します。

保存されるもの:

- テキスト1
- テキスト2
- draft
- カード一覧
- 区切り文字
- 本文編集ロック状態
- アクティブなテキストペイン
- 作成日時・更新日時
- アプリバージョン

「作業を開く」で復元できます。

## 著作権・プライバシー

このツールは、読み込んだ本文を外部サーバーへ送信しません。ブラウザ内で処理します。

ただし、著作権のある本文を使って作った問題ファイルを再配布・公開することは避けてください。個人学習用として使う前提で管理してください。

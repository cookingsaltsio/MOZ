# モンスターおぼえるぜ

「モンスターおぼえるぜ」は、単語や問題をモンスターとのバトル形式で学習するアプリです。

PCブラウザ、Android Chrome、iOS Safari、Android APK版で使うことを想定しています。学習データは端末内に保存されるため、アカウント登録なしで使えます。

## 主な機能

- デッキを読み込んで問題演習
- 間隔反復モード（SRS）による復習
- 全項目での出陣
- 項目一覧からの個別確認
- ミスした問題の復習
- 必殺技、回復スキル、クリティカル、コンボなどの演出
- 対戦IDを使った対戦
- 学習データのエクスポート / インポート
- 音量、ミュート、演出ON/OFFなどの設定

## 使い方

### ブラウザで使う場合

GitHub Pagesなどで公開されたページを開くと、そのままブラウザで使えます。

対戦まで使う場合は、`file://` で直接開くよりも、HTTPSで公開されたページか、ローカルサーバー経由で開くほうが安定します。

### Android APK版で使う場合

APKファイルを受け取ってAndroid端末にインストールすると、ブラウザを開かずに使えます。

Androidの設定によっては、提供元不明のアプリのインストール許可が必要になる場合があります。

## 学習データの保存

学習データは、各端末・各ブラウザの中に保存されます。

- PCブラウザとスマホブラウザの学習データは自動同期されません。
- Android APK版とブラウザ版の学習データも自動同期されません。
- 端末を変える場合は、エクスポート / インポートでデータを移してください。
- ブラウザのデータ削除、キャッシュ削除、アプリ削除を行うと、保存データが消える場合があります。

大事な学習データは、定期的にエクスポートしてバックアップしてください。

## 対応しているデッキ形式

読み込める形式は次の通りです。

- `txt`
- `csv`
- `tsv`
- `xlsx`
- `ods`
- `zip`

`apkg` には現在対応していません。

## デッキの書き方

### txt

1行を1カードとして読み込みます。

問題と解答は、タブ、`::`、2個以上の空白、または最初の空白で区切れます。

```text
cat 猫
dog 犬
apple::りんご
```

AnkiのCloze風の穴埋め形式にも対応しています。

```text
The capital of Japan is {{c1::Tokyo}}.
```

### csv / tsv / xlsx / ods

1行を1カードとして読み込みます。

- 1列目: 問題
- 2列目以降: 解答
- `choice_exclude_words` 列がある場合: 指定した語は不正解選択肢から除外されます
- `choice_group` 列がある場合: 同じグループのカードは不正解選択肢から除外されます

例:

```csv
question,answer,choice_exclude_words,choice_group
一丝一毫 yī sī yī háo,一点儿，极少或极小。,一针一线、一星半点、微乎其微,
一针一线 yī zhēn yī xiàn,比喻极微小的东西。,一丝一毫,
大同小异 dà tóng xiǎo yì,大体相同，略有差异。,,sim_test_001
异曲同工 yì qǔ tóng gōng,不同做法收到同样效果。,,sim_test_001
```

ヘッダー行が `question,answer`、`front,back`、`問題,解答` などの場合は自動でスキップします。

`choice_exclude_words` と `choice_group` は任意です。`choice_exclude_words` は `、`、`,`、`;`、`/`、改行、空白などで複数指定できます。

問題列が `一丝一毫 yī sī yī háo` のように「成語 + 半角スペース + pinyin」の場合、選択肢除外の比較では先頭の `一丝一毫` だけを使います。`choice_exclude_words` と `choice_group` 列は解答欄には混ざりません。

### zip

`zip` の中に、複数のデッキファイルをまとめて入れられます。

対応している中身:

- `txt`
- `csv`
- `tsv`
- `xlsx`
- `ods`
- モンスター画像用の画像ファイル

## 対戦について

対戦は PeerJS / WebRTC を使います。

基本的な流れは次の通りです。

1. 片方が部屋を作ります。
2. 表示された対戦IDを相手に送ります。
3. もう片方が対戦IDを入力して参加します。
4. ホストがデッキや対戦設定を決めます。
5. 問題セットを同期します。
6. 両方が準備OKになったら対戦を開始します。

対戦では、ホストが選んだ問題セットが相手の端末へ送信されます。通常の学習記録全体やSRS記録全体は送信されません。

通信環境やブラウザによっては接続に失敗する場合があります。その場合は、接続し直す、Chrome系ブラウザを使う、通信ログ内のトラブル対応項目を確認する、といった方法を試してください。

## 対応環境の目安

- PC: 最新版のChrome、Edge、Firefoxなど
- Android: Android Chrome、またはAPK版
- iPhone / iPad: iOS Safari

ブラウザ版では、ZIP読み込み、xlsx/ods読み込み、対戦機能に外部ライブラリやブラウザ機能を使います。通信制限や広告ブロックなどにより外部CDNを読み込めない場合、一部機能が使えないことがあります。

### GitHub Pagesでのプレイ

```text
https://cookingsaltsio.github.io/MOZ/
```

このリポジトリをGitHub Pagesで公開する場合、公開対象はリポジトリ直下の `index.html` です。

Android APK版で使う本体HTMLは `app/src/main/assets/index.html` です。ブラウザ公開版では、iPhone Safariなどで転送ページが止まる問題を避けるため、同じ内容をリポジトリ直下の `index.html` にも置いています。

アプリ本体を修正した場合は、公開前に `app/src/main/assets/index.html` の内容を直下の `index.html` にも反映してください。
